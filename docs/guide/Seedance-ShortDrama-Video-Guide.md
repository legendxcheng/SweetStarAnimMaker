# Seedance 2.0 短剧图生视频实战手册

## 目的

这份文档面向短剧视频操作者，解决一个非常具体的问题：

- 你有一个 `15 秒` 或更短的视频需求
- 这段视频通常包含 `2 到 5 个子镜头`
- 你希望尽量让 `Seedance 2.0` 一键生成这一整段
- 这段视频不是孤立素材，而是长视频中的一个中间片段
- 它必须和前后片段保持连续，不跳人、不跳情绪、不跳动作、不跳场景

本文不是 API 说明书，也不是泛泛的 prompt 入门。重点是给你一套可以直接照着写的工作方法。

## 适用场景

本文默认你做的是以下类型的短剧片段：

- 对话推进段
- 情绪反应段
- 信息揭示段
- 人物走位和进出场段
- 小冲突或关系推进段
- 上一段和下一段之间的承上启下段

如果你做的是强动作、大场面、极多角色同时调度、极密集剪辑的片段，单条 prompt 一键生成的成功率会显著下降，优先考虑拆段。

## 先看结论

对于短剧制作，`Seedance 2.0` 最稳的使用姿势不是“把所有镜头想法全塞进去”，而是：

1. 先判断这一段是否适合一条 prompt 生成
2. 再把这一段写成一条连续时间轴，而不是一串分裂镜头
3. 先锁定连续性，再追求风格
4. 参考图负责“长什么样”，prompt 负责“这几秒发生什么”
5. 片段结尾必须为下一段留出明确的接续状态

## 先统一参考素材语义

当前短剧工作流里，视频阶段不要再把所有图片都理解成同一种“参考图”。

更稳的理解方式是：

- `character_sheet`
  - 作用是锁定角色外观、服装、发型、体型和整体人设一致性
- `scene_sheet`
  - 作用是锁定环境空间、时代氛围、关键陈设和场景辨识度
  - 正式来源应优先是已审核的上游 `scene_sheet`
  - 如果该段没有可复用核心场景，才在视频阶段临时补场景参考
- `start_frame`
  - 作用是锁定当前 segment 的开场构图、动作起点和镜头起势
- `end_frame`
  - 作用是锁定当前 segment 的稳定终态，只在明确需要时使用

也就是说：

- `视频生成最小必需素材 = 角色设定参考图 + 首帧`
- `增强控制素材 = 场景设定图 + 尾帧`

这份手册里的“场景设定图”，默认优先指已经审核过的 `scene_sheet`，而不是视频阶段临时找的一张环境图。

## 官方已确认的关键边界

以下内容来自我在 `2026-04-17` 查验的火山引擎官方文档。

- `Seedance 2.0` 和 `Seedance 2.0 fast` 支持图生视频首帧、图生视频首尾帧、多模态参考生视频。
- 多模态参考生视频支持 `1 到 9` 张参考图片、`0 到 3` 段参考视频、`0 到 3` 段参考音频，但音频不能单独输入，至少要有图片或视频。
- 图生视频首尾帧模式与多模态参考模式是互斥场景，不能直接混用。
- `Seedance 2.0` 时长支持 `4 到 15` 秒，也支持 `duration = -1` 让模型自行决定长度。短剧制作里不建议交给模型自行决定。
- `Seedance 2.0` 支持 `adaptive` 比例，但如果输出比例和输入图片比例不一致，官方会裁剪，而且是居中裁剪。当前短剧横屏主链路必须显式约束为 `ratio = "16:9"`，不要依赖 `adaptive`。
- `Seedance 2.0` 支持返回尾帧图像，可把上一段生成结果的尾帧作为下一段的首帧，快速生成连续片段。
- 官方提示词指南明确建议对多图参考使用 `图片1 / 图片2 / ...` 这种显式编号指代。
- 官方 API 文档明确写明：`Seedance 2.0` 系列不支持直接上传含真人人脸的参考图或参考视频。涉及真人肖像时，要走平台允许的素材或虚拟人像方案。

这几个边界直接决定了短剧片段的做法：你不能只靠“多图参考”幻想严格控制首尾状态；要做片段接力时，首尾帧和返回尾帧能力很重要。

本仓库 worker 已在 Seedance 视频 provider 配置入口默认传 `SEEDANCE_ASPECT_RATIO = "16:9"` 的等效值。只有当你明确要做竖屏短剧或平台切版时，才把环境变量改成 `9:16`；否则方图首帧、多图参考和 `adaptive` 组合很容易生成非横屏视频。

## 一条 prompt 什么时候适合做完整短剧片段

不是所有 `15 秒` 片段都应该一键生成。

适合一条 prompt 做完整片段的情况：

- 这一段只有 `一条明确情绪线`
- 这一段只有 `一条明确动作线`
- 内部虽然有多个子镜头，但它们本质上是同一事件的连续展开
- 场景没有发生根本切换
- 人物数量可控，最好不超过 `2 到 3` 个主要行动角色

不适合一条 prompt 直接做的情况：

- 同一段里既要强情绪爆发，又要复杂走位，又要高强度动作
- 子镜头之间本来就依赖大量硬切或蒙太奇
- 片段内部跨越多个场景
- 你同时要求多次大角度机位切换和强剧情推进
- 这一段本身就应该拆成两个叙事功能不同的小段

### 快速判断法

先问自己一句：

`这 15 秒里，到底只完成一个什么事件变化？`

如果你不能把这段压成一句清晰的话，例如：

- 她听完这句话，先愣住，再强装镇定，最后转身离开
- 他进门发现空无一人，环顾四周，最后视线停在桌上的手机
- 两人对视，女方步步逼近，男方后退到墙边

那就先不要一键做整段，先拆。

## Seedance 短剧片段 prompt 的推荐结构

短剧片段最稳的写法，不是“主体 + 动作 + 风格”这么简单，而是下面这套连续性优先结构：

```text
【片段定位】
这是长视频中的中间片段，承接上一段的[状态]，并为下一段的[状态]做准备。

【连续性约束】
保持[人物外观/服装/场景/道具/时间氛围]一致。
本段中不要改变[发型/服装颜色/房间陈设/人物关系状态/时间段]。

【开头承接状态】
片段开始时，[人物A]已经处于[动作/情绪/站位]，[人物B]处于[动作/情绪/站位]。
画面延续上一段已建立的[构图/机位方向/空间关系]。

【子镜头时间轴】
0-3秒：
[当前镜头看到什么]，[人物做什么]，[镜头怎么动]。

3-7秒：
[上一小段如何自然接过来]，[动作如何继续]，[镜头如何变化]。

7-11秒：
[情绪或信息如何推进]，[镜头如何继续服务推进]。

11-15秒：
[本段收束动作]，[镜头落点]，[为下一段保留什么状态]。

【结尾交接状态】
片段结束时，人物停在[动作/眼神/站位/情绪]，镜头停在[构图]，便于下一段直接继续。

【风格与限制】
整体风格为[写实/电影感/生活流/克制写实]。
镜头语言保持[平稳/轻微跟拍/缓慢推近]，不要过度炫技，不要加入额外角色和额外情节。
```

## 这套结构为什么适合短剧

因为短剧制作的真正难点不是“生成一个好看视频”，而是“生成一个可剪进去的视频”。

对于中间片段，你必须同时控制：

- 这一段从哪里接进来
- 这一段内部怎么推进
- 这一段落在哪里

如果你的 prompt 只写中间动作，不写开头承接状态和结尾交接状态，这段视频即使单看不错，也很容易接不上前后片段。

## 多个子镜头怎么写进一条 prompt

### 原则

把多个子镜头写成 `连续时间轴`，不要写成互相独立的分镜清单。

错误写法：

```text
先特写，再中景，再俯拍，再切手部特写，再空镜，再人物回头，再特写眼泪。
```

这种写法的问题是：

- 没有说明每一段为什么切
- 没有说明动作如何连续
- 模型容易把它理解为分裂的镜头愿望清单

更好的写法：

```text
0-4秒：中景承接上一段，她站在门口停住，眼神发怔，镜头缓慢推近。
4-8秒：她听见屋内声音后回神，轻轻抬头看向右侧，镜头跟着她的视线轻微横移。
8-12秒：切近景，她强压情绪，嘴角勉强扬起，随后转身。
12-15秒：镜头留在她转身后的侧背身，她停顿半秒，像是准备离开，为下一段接她迈步离开做准备。
```

### Seedance 更适合的子镜头关系

推荐：

- 推近
- 轻微拉远
- 跟拍
- 小幅横移
- 中景到近景
- 同场景内的视线转移
- 同一动作链里的自然切换

谨慎使用：

- 频繁大角度切机位
- 明显不连续的俯拍、顶拍、反打混用
- 一条 prompt 内做强 montage
- 一条 prompt 内同时做人物戏、空镜、象征镜头、回忆插入

短剧里最稳的是“一个机位主线 + 少量服务叙事的变化”。

## 作为长视频中间片段，连续性到底要写什么

很多人说“保持一致性”，但这句话对操作者没有帮助。你真正要写的是下面这些具体维度。

### 1. 人物外观连续

必须明确不变项：

- 发型
- 服装
- 配饰
- 年龄感
- 妆面和疲态

可直接写：

```text
保持人物脸部特征、发型、白色衬衫、深色外套和耳饰一致。
```

### 2. 情绪连续

短剧最常见的问题不是脸漂，而是情绪断。

你要写的是：

- 这一段开头时，人物已经处于什么情绪
- 这一段里情绪如何变化
- 结尾时情绪停在哪里

可直接写：

```text
片段开始时她已经处于强压委屈、表面冷静的状态，片段内情绪从僵住到勉强镇定，结尾停在决心离开的克制状态。
```

### 3. 动作起点连续

如果上一段结尾是“他刚推门进来”，那这一段开头就不能又让他从门外重新走进来。

你要写：

```text
片段开始时他已经推门进入室内半步，右手还停在门把上。
```

### 4. 动作终点连续

这决定下一段能不能接。

你要写：

```text
片段结尾停在她已经转身但尚未迈出第一步的状态，便于下一段直接接她离开。
```

### 5. 场景与空间关系连续

不要只写“在客厅里”。要写人物站位和空间方向。

例如：

```text
延续上一段已建立的客厅空间关系，女生靠近餐桌左侧，男生站在入户门附近，两人之间保持对峙距离。
```

### 6. 镜头进入点和落点连续

这是最容易被忽略的一层。

你要告诉模型：

- 本段开头镜头大概从哪里接进来
- 本段结尾镜头停在哪

例如：

```text
镜头延续上一段的中景视角进入，不重新建立全景。结尾落在女生侧背身的中近景。
```

## 参考图怎么喂，才适合短剧片段

### 情况 1：主要目标是保人物

优先准备：

- 正脸图
- 三分之二侧脸图
- 半身图

如果角色外观一致性比镜头复杂度更重要，先保证人物图足够清晰，再写 prompt。

### 情况 2：同时要保人物、服装、场景

不要幻想一张图全解决。更稳的方式是拆图：

- `图片1` 人物
- `图片2` 服装或造型关键细节
- `图片3` 场景
- `图片4` 道具或标志性元素

然后明确写：

```text
图片1中的女生穿图片2中的服装，位于图片3中的客厅环境里，桌上的手机保持图片4中的外观。
```

这里的 `图片3`，如果你的项目已经有正式 `scene_sheet`，优先直接使用它，不要重新临时找一张无约束的环境图顶上。

### 情况 3：你要做的是中间片段接力

这里最重要的是首尾衔接，不只是参考图。

优先级建议：

1. 如果必须严格控制开头和结尾，优先使用 `首尾帧模式`
2. 如果更在意多个元素的综合参考，可考虑 `多模态参考生视频`
3. 如果你已经生成了上一段，优先拿上一段的尾帧做下一段首帧

官方 API 支持返回尾帧图像，这对连续短剧片段特别有用。实操上可以形成这种链路：

```text
上一段视频 -> 返回尾帧 -> 作为当前段首帧 -> 当前段生成 -> 返回尾帧 -> 作为下一段首帧
```

这样比只靠“参考图 + 文字说明”更稳。

## 多图参考时的写法

官方已经明确建议用显式编号。

推荐写法：

```text
参考图片1中的女生形象，保持脸部特征一致。
她穿着图片2中的米色风衣，站在图片3中的医院走廊里。
```

不推荐写法：

```text
参考上传的几张图。
```

后者对短剧制作几乎没有可控性。

## 与前后片段保持一致的最小检查表

在你发起生成前，至少检查这 7 项：

- 这一段开头人物已经处于什么动作
- 这一段开头人物已经处于什么情绪
- 这一段场景空间关系是否和上一段一致
- 这一段中绝对不能变化的元素是什么
- 这一段内部推进的是哪一个事件变化
- 这一段结尾人物停在什么状态
- 这一段结尾镜头是否能给下一段直接接上

如果这 7 项里你有 2 项以上说不清，就不要急着生成。

## 操作者最常用的主模板

下面这条模板是给短剧中间片段准备的，可以直接改。

```text
这是长视频中的中间片段，承接上一段已经建立的状态，不要重新介绍场景，不要新增无关人物或无关情节。

保持图片1中的[人物A]外观一致，保持图片2中的[人物B]外观一致，保持图片3中的[场景]一致。服装、发型、道具、时间氛围保持不变。

片段开始时，[人物A]已经处于[开头动作与情绪]，[人物B]处于[开头动作与情绪]，两人的站位关系为[站位关系]。镜头延续上一段的[镜头进入点]，不要重新建立大全景。

0-3秒：[子镜头1的画面、动作、镜头]
3-7秒：[子镜头2的画面、动作、镜头]
7-11秒：[子镜头3的画面、动作、镜头]
11-15秒：[子镜头4的画面、动作、镜头]

这一段只完成[一句话事件推进]。整体风格保持[写实/电影感/生活流/克制]，镜头语言以[平稳/轻微跟拍/缓慢推近]为主，不要炫技，不要过度切镜。

片段结尾时，[人物A/人物B]停在[结尾动作、结尾情绪、结尾站位]，镜头落在[结尾构图]，便于下一段直接承接。
```

## 四类高频短剧模板

### 1. 对话推进段

适合：

- 两人说话
- 情绪逐渐变化
- 需要一个可接前后段的中间对话片段

模板：

```text
这是对话推进段，承接上一段两人已经开始交谈的状态。
保持图片1中的女生和图片2中的男生外观一致，保持图片3中的办公室环境一致。

片段开始时女生已经看着男生，表面平静但明显压着情绪，男生站在她对面略显迟疑。镜头延续上一段的中景，不重新建立环境。

0-4秒：中景承接，女生先开口，语气克制，男生沉默看着她，镜头轻微推近。
4-8秒：切男生近景，他听完后眼神躲闪，张嘴想解释又停住。
8-12秒：切回女生中近景，她勉强笑了一下，但笑意很冷。
12-15秒：镜头停在女生侧脸近景，她说完最后一句后移开视线，留下压抑停顿，为下一段接男生反应做准备。

这一段只推进“女生挑明问题，男生开始动摇”这一件事。
```

### 2. 情绪反应段

适合：

- 听到消息后的僵住、回神、压情绪
- 片段重点不在对话，而在反应变化

模板：

```text
这是情绪反应段，承接上一段她刚听到坏消息后的瞬间。
保持图片1中的女生外观和图片2中的病房环境一致，不增加额外人物。

片段开始时她已经站在病房门口，身体僵住，眼神失焦。镜头从上一段留下的中近景直接接入。

0-5秒：镜头缓慢推近，她短暂失神，呼吸发紧，手指微微收紧。
5-10秒：她回过神来，视线慢慢落到病床方向，眼眶发红但不落泪。
10-15秒：她强行稳住情绪，轻轻吸一口气，转身准备离开，镜头停在她侧背身，为下一段接她出门做准备。

这一段只完成“从失神到强行镇定”这一情绪变化。
```

### 3. 信息揭示段

适合：

- 发现手机、照片、文件、证据
- 需要把注意力从人物转移到关键物件

模板：

```text
这是信息揭示段，承接上一段男生进入房间后的搜索状态。
保持图片1中的男生、图片2中的房间环境、图片3中的手机外观一致。

片段开始时男生已经站在桌边，视线正在扫过桌面。镜头承接上一段的中景进入。

0-4秒：中景里他快速扫视桌面，随后注意力停在桌角。
4-8秒：镜头跟着他的视线轻微下移，看到图片3中的手机。
8-12秒：切手部和手机的近景，他伸手拿起手机，表情明显变化。
12-15秒：镜头落在他拿着手机的近景，他盯着屏幕僵住，为下一段接屏幕内容或他的反应做准备。

这一段只推进“他发现了关键信息”。
```

### 4. 承上启下过渡段

适合：

- 上一段已经有情绪爆点
- 当前段要把人物送到下一段的动作起点

模板：

```text
这是承上启下过渡段，承接上一段争执结束后的余波，为下一段她离开房间做准备。
保持图片1中的女生和图片2中的客厅环境一致，维持夜晚室内光线和当前服装状态不变。

片段开始时女生已经背对男生，站在沙发前停住，情绪强压但没有回头。镜头从上一段结束时的侧后方中景直接接入。

0-4秒：她站着不动，只能看到肩膀轻微起伏，空气压抑，镜头缓慢推近。
4-9秒：她慢慢抬手擦掉快要落下的眼泪，没有回头。
9-12秒：她深吸一口气，身体重心前移，像是终于做出决定。
12-15秒：她抬脚准备离开，镜头留在她侧背身中景，便于下一段直接接她走出客厅。

这一段只负责把情绪爆点过渡到离开动作。
```

## 如何针对 Seedance 做更稳的限制

短剧 prompt 里要学会写“不允许什么”。

推荐直接加：

- 不要新增无关角色
- 不要改变服装和场景陈设
- 不要重新建立全景
- 不要加入夸张特效
- 不要做过度炫技运镜
- 不要在中间加入无关空镜

这类限制对短剧特别有用，因为短剧片段最怕模型自己“加戏”。

## 常见失败与修正方法

### 1. 人物外观漂了

表现：

- 脸变了
- 发型变了
- 服装局部变了

优先修改：

- 增加更清晰的角色参考图
- 在 prompt 里明确写“不变项”
- 把风格词和动作词减掉一些，避免模型把注意力分散

先改这里，不要先加更多镜头词。

### 2. 情绪接不上上一段

表现：

- 上一段还在哭，这一段一开头已经平静
- 上一段在压抑，这一段突然变得很外放

优先修改：

- 补 `开头承接状态`
- 明确写“片段开始时人物已经处于什么情绪”
- 把这一段的情绪变化写成一条线，而不是多次来回波动

### 3. 子镜头之间太碎，像胡乱拼接

表现：

- 镜头变化很多，但事件没有推进
- 画面像剪了很多不连续片段

优先修改：

- 减少子镜头数量
- 每段只保留一个主要动作推进
- 把镜头变化改成连续动作链，而不是独立镜头列表

短剧一条 `15 秒` prompt 里，通常 `3 到 4` 个子镜头最稳。

### 4. 和前一段接不上

表现：

- 重复了上一段已经完成的动作
- 开头机位完全变了
- 场景关系重新洗牌

优先修改：

- 明确写“片段开始时人物已经处于什么动作”
- 写“镜头延续上一段的什么视角进入”
- 禁止重新建立环境全景

### 5. 和下一段接不上

表现：

- 当前段结尾停得太死
- 结尾动作已经做完，下一段无从接续

优先修改：

- 补 `结尾交接状态`
- 让结尾停在“动作即将继续”的状态上
- 把结尾构图写明确

短剧里一个好结尾通常不是“动作彻底结束”，而是“下一动作马上能开始”。

### 6. 运镜太花，抢戏

表现：

- 情绪戏却出现大量炫技运动
- 角色表演被镜头存在感压过去

优先修改：

- 把运镜词收敛到 `缓慢推近 / 轻微跟拍 / 小幅横移`
- 删除无必要的高角度、俯拍、旋转、复杂运动
- 重申“以人物表演和情绪推进为主”

### 7. 场景总是漂

表现：

- 同一段里房间结构换了
- 关键陈设、时代氛围或环境辨识度不稳定
- 前后片段像发生在两个不同地方

优先修改：

- 优先改用已审核 `scene_sheet`
- 在 prompt 里明确写空间结构、关键陈设和禁止漂移项
- 如果该段不属于任何正式核心场景，再补临时场景参考

不要一上来就加更多镜头词，先把场景锚点补稳。

## 一套推荐工作流

如果你真的要把一个中间片段做稳，建议按这个顺序：

1. 先明确本段只推进哪一个事件变化
2. 写本段开头承接状态
3. 写本段结尾交接状态
4. 再把中间拆成 `2 到 5` 个子镜头时间段
5. 再补参考图编号和不变项
6. 最后才补风格和镜头语言

这和很多人直觉相反。多数人先写风格，但短剧生产里最重要的是接得上。

## 一句最重要的判断标准

如果你的 prompt 读完之后，别人仍然说不出：

- 这段从哪里接进来
- 这段中间推进了什么
- 这段最后停在哪里

那这条 prompt 还没有写好。
