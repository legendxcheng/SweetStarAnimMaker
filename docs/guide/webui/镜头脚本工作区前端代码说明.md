# 镜头脚本工作区前端代码说明

## 先看结论

如果你要优化【镜头脚本工作区】前端，最先看的不是一个文件，而是两层页面：

1. `apps/studio/src/components/shot-script-phase-panel.tsx`
2. `apps/studio/src/pages/shot-script-review-page.tsx`

区别很重要：

- `shot-script-phase-panel.tsx` 是项目详情页里的“镜头脚本工作区”摘要页
- `shot-script-review-page.tsx` 是真正的镜头脚本审核 / 编辑页

所以：

- 只改项目详情里的展示卡片、详情只读布局、跳转按钮，优先改 `shot-script-phase-panel.tsx`
- 改段落编辑、镜头字段表单、保存 / 通过 / 重生成交互，优先改 `shot-script-review-page.tsx`

---

## 页面入口和调用关系

### 1. 项目详情页入口

`apps/studio/src/pages/project-detail-page.tsx`

这个文件负责：

- 定义左侧阶段导航里的 `镜头脚本`
- 判断什么时候允许进入镜头脚本阶段
- 决定什么时候默认选中镜头脚本阶段
- 把 `project`、`task`、`creatingTask`、`onGenerate` 传给 `ShotScriptPhasePanel`

与镜头脚本工作区直接相关的点：

- `getDefaultProjectPhase()`
- `isShotScriptPhaseEnabled()`
- `canRegenerateShotScript()`
- `handleRegenerateShotScript()`

如果你要改：

- `镜头脚本` 阶段是否可点击
- 阶段默认跳转
- 顶部 `重新生成` 的行为

就要改这个文件。

### 2. 项目详情页中的镜头脚本工作区

`apps/studio/src/components/shot-script-phase-panel.tsx`

这个文件负责：

- 渲染“镜头脚本工作区”标题和说明
- 渲染顶部 `重新生成` 按钮
- 展示任务状态
- 展示当前镜头脚本摘要
- 通过 `apiClient.getCurrentShotScript(project.id)` 拉取完整镜头脚本详情
- 把镜头脚本按 Segment 和 Shot 只读展开
- 在 `shot_script_in_review` 时显示“进入镜头脚本审核”

这是“概览页 + 只读详情页”，不是主要编辑页。

### 3. 镜头脚本审核 / 编辑页

`apps/studio/src/pages/shot-script-review-page.tsx`

这个文件是真正的镜头脚本 webui 工作台，负责：

- 加载 `apiClient.getShotScriptReviewWorkspace(projectId)`
- 建立每个 Segment 的本地草稿 `drafts`
- 跟踪脏数据 `dirtySegmentIds`
- 编辑 Segment 标题、摘要、时长
- 编辑每个 Shot 的所有字段
- 保存单段
- 通过单段
- 重生成单段
- 全部通过
- 整体重新生成

如果你要改的是：

- 审核页布局
- 表单字段
- 段落卡片结构
- Shot 级编辑器
- 保存 / 通过 / 重生成按钮逻辑

核心文件就是这个。

## 现在这条前端链路怎么走

### A. 项目详情里的镜头脚本工作区

1. 用户进入 `project-detail-page.tsx`
2. 页面调用 `apiClient.getProjectDetail(projectId)`
3. 页面根据项目状态决定是否启用 `镜头脚本`
4. 选中后渲染 `ShotScriptPhasePanel`
5. `ShotScriptPhasePanel` 读取 `project.currentShotScript`
6. 如果存在当前镜头脚本，再调用 `apiClient.getCurrentShotScript(project.id)` 拉完整详情
7. 页面按 Segment -> Shot 层级只读展开

### B. 镜头脚本审核页

1. 用户从详情页点击“进入镜头脚本审核”
2. 进入 `apps/studio/src/pages/shot-script-review-page.tsx`
3. 页面调用 `apiClient.getShotScriptReviewWorkspace(projectId)`
4. 返回数据被转换成 `drafts`
5. 用户修改 Segment 或 Shot 字段
6. 页面把修改记录到本地 state，并标记脏段落
7. 用户执行单段保存、单段通过、单段重生成，或顶部全部通过 / 整体重生成

## 常见改动场景，对应改哪些文件

### 1. 只改项目详情页里的镜头脚本展示

优先改：

- `apps/studio/src/components/shot-script-phase-panel.tsx`

必要时补：

- `apps/studio/src/pages/project-detail-page.tsx`

### 2. 改镜头脚本审核页的编辑表单

改：

- `apps/studio/src/pages/shot-script-review-page.tsx`

如果字段变了，再联动：

- `packages/shared/src/types/shot-script.ts`
- `packages/shared/src/schemas/shot-script-api.ts`
- `apps/studio/src/services/api-client.ts`

### 3. 想给 Shot 新增一个字段

至少要看：

- `packages/shared/src/types/shot-script.ts`
- `packages/shared/src/schemas/shot-script-api.ts`
- `apps/studio/src/pages/shot-script-review-page.tsx`
- `apps/studio/src/components/shot-script-phase-panel.tsx`
- `apps/studio/src/services/api-client.ts`

### 4. 想改单段保存 / 通过 / 重生成逻辑

改：

- `apps/studio/src/pages/shot-script-review-page.tsx`
- `apps/studio/src/services/api-client.ts`

如果段落选择规则变化，再看：

- `packages/shared/src/shot-script-segment-selector.ts`

### 5. 想改顶部整体重新生成按钮

详情页顶部按钮：

- `apps/studio/src/pages/project-detail-page.tsx`

审核页顶部按钮：

- `apps/studio/src/pages/shot-script-review-page.tsx`

接口层：

- `apps/studio/src/services/api-client.ts`

### 6. 想改“进入镜头脚本审核”的入口或 review route

改：

- `apps/studio/src/components/shot-script-phase-panel.tsx`
- 对应路由注册文件

如果只改入口按钮文案或显隐，通常先改 `shot-script-phase-panel.tsx`。

## 一句话总结

要优化【镜头脚本工作区】前端，摘要和只读展示主要改 `apps/studio/src/components/shot-script-phase-panel.tsx`，真正的审核编辑体验主要改 `apps/studio/src/pages/shot-script-review-page.tsx`；如果字段或接口变动，再同步联动 `api-client.ts`、`packages/shared/src/types/shot-script.ts`、`packages/shared/src/schemas/shot-script-api.ts` 和 `packages/shared/src/shot-script-segment-selector.ts`。
