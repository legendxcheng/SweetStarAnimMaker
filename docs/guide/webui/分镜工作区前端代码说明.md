# 分镜工作区前端代码说明

## 先看结论

如果你要优化【分镜工作区】前端，最核心的两层文件是：

1. `apps/studio/src/components/storyboard-phase-panel.tsx`
2. `apps/studio/src/pages/review-workspace-page.tsx`

区别：

- `storyboard-phase-panel.tsx` 是项目详情页里的“分镜工作区”摘要页和只读详情页
- `review-workspace-page.tsx` 是真正的分镜审核 / 编辑页

所以：

- 只改项目详情中的分镜概览、只读详情、进入审核按钮，优先改 `storyboard-phase-panel.tsx`
- 改分镜审核编辑器、保存 / 通过 / 驳回 / 重新生成交互，优先改 `review-workspace-page.tsx`

---

## 页面入口和调用关系

### 1. 项目详情页入口

`apps/studio/src/pages/project-detail-page.tsx`

这个文件负责：

- 定义左侧阶段导航里的 `分镜`
- 判断分镜阶段何时可点击
- 控制默认是否切到分镜阶段
- 把 `project`、`task`、`creatingTask`、`onGenerate` 传给 `StoryboardPhasePanel`

和分镜工作区直接相关的点：

- `getDefaultProjectPhase()`
- `isStoryboardPhaseEnabled()`
- `canRegenerateStoryboard()`
- `handleRegenerateStoryboard()`

### 2. 项目详情页中的分镜工作区

`apps/studio/src/components/storyboard-phase-panel.tsx`

这个文件负责：

- 渲染“分镜工作区”标题、说明和顶部 `重新生成`
- 展示任务状态
- 展示当前分镜摘要
- 通过 `apiClient.getCurrentStoryboard(project.id)` 拉取完整分镜详情
- 以 Scene -> Segment 形式只读展开
- 在 `storyboard_in_review` 时展示“进入分镜审核”

它是分镜的只读入口页，不是主要编辑器。

### 3. 分镜审核 / 编辑页

`apps/studio/src/pages/review-workspace-page.tsx`

虽然文件名比较泛，但它实际就是当前的分镜审核工作台。这个文件负责：

- 调用 `apiClient.getStoryboardReviewWorkspace(projectId)`
- 维护可编辑草稿 `draft`
- 跟踪 `hasChanges`
- 编辑标题、集标题、场景名称、戏剧目的
- 编辑每个 Segment 的画面、动作、对白、旁白、音频、目的、时长
- 保存修改
- 通过分镜
- 驳回分镜
- 重新生成分镜

如果你要改真正的分镜编辑体验，优先改这个文件。

## 现在这条前端链路怎么走

### A. 项目详情里的分镜工作区

1. 用户进入 `project-detail-page.tsx`
2. 页面调用 `apiClient.getProjectDetail(projectId)`
3. 页面根据状态判断是否启用 `分镜`
4. 选中后渲染 `StoryboardPhasePanel`
5. `StoryboardPhasePanel` 读取 `project.currentStoryboard`
6. 如果存在当前分镜，再调用 `apiClient.getCurrentStoryboard(project.id)` 拉完整详情
7. 页面以 Scene -> Segment 的层级只读展示

### B. 分镜审核页

1. 用户点击“进入分镜审核”
2. 进入 `apps/studio/src/pages/review-workspace-page.tsx`
3. 页面调用 `apiClient.getStoryboardReviewWorkspace(projectId)`
4. 工作区数据转换成可编辑 `draft`
5. 用户编辑标题、场景和段落字段
6. 页面用 `hasChanges` 控制保存 / 通过 / 驳回 / 重新生成按钮状态
7. 用户保存、通过、驳回或重新生成后返回项目详情页

## 常见改动场景，对应改哪些文件

### 1. 只改项目详情页里的分镜概览和详情展示

优先改：

- `apps/studio/src/components/storyboard-phase-panel.tsx`

必要时补：

- `apps/studio/src/pages/project-detail-page.tsx`

### 2. 改分镜审核页编辑表单

改：

- `apps/studio/src/pages/review-workspace-page.tsx`

如果字段变了，再联动：

- `packages/shared/src/types/storyboard.ts`
- `packages/shared/src/schemas/storyboard-api.ts`
- `apps/studio/src/services/api-client.ts`

### 3. 想给 Scene 或 Segment 新增字段

至少要看：

- `packages/shared/src/types/storyboard.ts`
- `packages/shared/src/schemas/storyboard-api.ts`
- `apps/studio/src/pages/review-workspace-page.tsx`
- `apps/studio/src/components/storyboard-phase-panel.tsx`
- `apps/studio/src/services/api-client.ts`

### 4. 想改通过 / 驳回 / 重新生成逻辑

改：

- `apps/studio/src/pages/review-workspace-page.tsx`
- `apps/studio/src/services/api-client.ts`

如果后端契约变化，再补：

- `packages/shared/src/schemas/storyboard-api.ts`

### 5. 想改项目详情页顶部“重新生成”按钮

改：

- `apps/studio/src/pages/project-detail-page.tsx`
- `apps/studio/src/components/storyboard-phase-panel.tsx`
- `apps/studio/src/services/api-client.ts`

### 6. 想改“进入分镜审核”的入口或路由

改：

- `apps/studio/src/components/storyboard-phase-panel.tsx`
- 对应路由注册文件

## 一句话总结

要优化【分镜工作区】前端，项目详情页里的概览和只读展示主要改 `apps/studio/src/components/storyboard-phase-panel.tsx`，真正的审核编辑体验主要改 `apps/studio/src/pages/review-workspace-page.tsx`；如果字段或接口结构变化，再同步联动 `api-client.ts`、`packages/shared/src/types/storyboard.ts` 和 `packages/shared/src/schemas/storyboard-api.ts`。
