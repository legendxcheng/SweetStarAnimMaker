# 画面工作区前端代码说明

## 先看结论

如果你要优化【画面工作区】前端页面，最核心的脚本是：

1. `apps/studio/src/components/image-phase-panel.tsx`

这个文件就是“画面工作区”的主体 UI，包含：

- 顶部摘要卡片
- 当前任务状态展示
- Segment 列表
- 起始帧 / 结束帧编辑卡
- Prompt 保存、重生成、出图、审核通过等按钮交互
- 画面批次与帧列表的自动刷新

如果只是改画面工作区的布局、文案、卡片结构、按钮顺序、字段展示，大多数情况下只需要先改这个文件。

---

## 页面入口和调用关系

### 1. 页面入口

`apps/studio/src/pages/project-detail-page.tsx`

这个文件负责：

- 定义项目阶段导航 `PROJECT_PHASES`
- 控制“画面”阶段是否可点击
- 决定默认选中哪个阶段
- 把 `project`、`task`、`creatingTask`、`onGenerate` 等参数传给 `ImagePhasePanel`

和画面工作区直接相关的点：

- 阶段名 `画面`
- `getDefaultProjectPhase()` 中是否自动切到 `images`
- `isImagesPhaseEnabled()` 中是否允许进入画面工作区
- `handleRegenerateImages()` 中顶部“重新生成”按钮触发的行为
- `onProjectRefresh={loadProject}` 这一层刷新回流

结论：

- 改“画面工作区是否可进入 / 默认是否跳到画面 / 顶部重新生成逻辑”时，改这个文件
- 改工作区内部卡片和交互时，优先改 `image-phase-panel.tsx`

### 2. 工作区主体

`apps/studio/src/components/image-phase-panel.tsx`

这是画面工作区主文件，当前职责很集中：

- 首次进入时调用 `apiClient.listImages(project.id)` 拉取帧列表
- 在 `images_generating` 或 `planStatus === "pending"` 时每 3 秒轮询
- 按 `sceneId + segmentId + order` 分组生成 Segment 卡片
- 在每个 Segment 下渲染起始帧和结束帧两个 `FrameEditorCard`
- 维护本地草稿 `drafts`
- 执行单帧和整批操作

当前这个组件内部已经同时承载了：

- 数据获取
- 数据轮询
- 草稿状态
- 操作按钮
- 卡片渲染

所以如果以后要做较大的前端优化，比如：

- 分拆子组件
- 增加图片预览
- 增加筛选 / 折叠 / 批量编辑
- 调整 Segment 分组方式
- 引入更细的 loading / empty / error 区块

通常第一步都应该从这个文件开始。

## 现在这页的数据流

可以按下面顺序理解：

1. 用户进入 `project-detail-page.tsx`
2. 页面通过 `apiClient.getProjectDetail(projectId)` 拉取项目详情
3. 页面判断当前阶段，并在选中 `images` 时渲染 `ImagePhasePanel`
4. `ImagePhasePanel` 检查 `project.currentImageBatch`
5. 如果存在当前批次，组件调用 `apiClient.listImages(project.id)` 拉取帧列表
6. 组件把帧数据分组成 Segment 卡片，再渲染起始帧 / 结束帧编辑区
7. 用户点击保存、重生成 Prompt、生成图片、审核通过时，组件直接调用对应 API
8. 某些操作完成后会触发：
   - `refreshFrames()` 重新拉取帧列表
   - `refreshProject()` 回到父页面刷新项目状态

特别注意：

- 顶部“重新生成”不是单帧操作，它来自父页面 `project-detail-page.tsx`
- 单帧 Prompt 和单帧图片相关动作，都在 `image-phase-panel.tsx` 内部处理

## 常见改动场景，对应改哪些文件

### 1. 只改页面排版、文案、卡片结构

优先改：

- `apps/studio/src/components/image-phase-panel.tsx`

可能顺带改：

- `apps/studio/src/styles/button-styles.ts`

### 2. 想给每帧增加新展示字段

至少要看：

- `packages/shared/src/types/shot-image.ts`
- `packages/shared/src/schemas/image-api.ts`
- `apps/studio/src/services/api-client.ts`
- `apps/studio/src/components/image-phase-panel.tsx`

如果字段来自项目详情摘要，还要看：

- `packages/shared/src/types/project-detail.ts`
- 项目详情对应 schema 文件

### 3. 想改“画面”阶段的启用条件或默认跳转逻辑

改：

- `apps/studio/src/pages/project-detail-page.tsx`

重点函数：

- `getDefaultProjectPhase()`
- `isImagesPhaseEnabled()`
- `canRegenerateImages()`

### 4. 想改顶部“重新生成”按钮的行为

改：

- `apps/studio/src/pages/project-detail-page.tsx`
- `apps/studio/src/services/api-client.ts`

如果后端契约变动，再补：

- `packages/shared/src/schemas/image-api.ts`
- 相关 shared 类型

### 5. 想改单帧按钮行为

改：

- `apps/studio/src/components/image-phase-panel.tsx`
- `apps/studio/src/services/api-client.ts`

单帧按钮包括：

- 保存提示词
- 重新生成 Prompt
- 生成图片
- 审核通过

### 6. 想改自动刷新 / 轮询逻辑

改：

- `apps/studio/src/components/image-phase-panel.tsx`
- `apps/studio/src/pages/project-detail-page.tsx`
- `apps/studio/src/hooks/use-task-polling.ts`

区别：

- `image-phase-panel.tsx` 负责画面帧列表自身的轮询
- `project-detail-page.tsx` 负责项目处于 `*_generating` 时的项目详情刷新
- `use-task-polling.ts` 负责有明确 taskId 时的任务轮询

### 7. 想新增图片预览、缩略图、灯箱

第一优先：

- `apps/studio/src/components/image-phase-panel.tsx`

同时确认：

- 后端是否已提供可直接访问的图片 URL
- 目前组件里展示的是 `imageAssetPath` 文本，不是实际图片组件

如果需要把相对路径转成可访问地址，还要看：

- `apps/studio/src/services/config.ts`
- 相关静态资源访问约定

要优化【画面工作区】前端页面，先从 `apps/studio/src/components/image-phase-panel.tsx` 下手；如果改动牵涉阶段切换、顶部重新生成、接口返回结构或字段契约，再继续联动 `project-detail-page.tsx`、`api-client.ts`、`packages/shared/src/types/shot-image.ts` 和 `packages/shared/src/schemas/image-api.ts`。
